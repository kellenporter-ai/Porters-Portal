# Rubric Format Reference

This document defines the exact rubric format used in Porters-Portal assessments. All generated rubrics MUST follow this structure precisely.

---

## 5-Level Grading Scale

Every rubric uses exactly these 5 levels with fixed percentages:

| Level | Percentage | Meaning |
|-------|-----------|---------|
| **Missing** | 0% | Nothing is submitted. |
| **Emerging** | 55% | Something is submitted, but the student has little to no ability to perform the assessed skill. |
| **Approaching** | 65% | Something is submitted, and the student has some ability to perform the assessed skill. |
| **Developing** | 85% | Something is submitted, and the student has the ability to perform the assessed skill with some minor difficulties. |
| **Refining** | 100% | Something is submitted, and the student has the ability to perform the assessed skill. The student needs to continue practicing the skill so they can maintain their ability. |

---

## Rubric Structure

### Organization

Each assessment question has its own rubric table. Each rubric table contains:

1. **A title row** — e.g., "MP3 Rubric: Assessment Question 1"
2. **Column headers** — `Skill | Missing 0% | Emerging 55% | Approaching 65% | Developing 85% | Refining 100%`
3. **One row per skill** — each skill being assessed gets its own row across all 5 levels

### Skill Statements

Each skill row begins with an "I am able to..." statement in the Skill column. These describe what the student should demonstrate. Examples:

- "I am able to read a bar chart which can track the quantity of something within a system of interest."
- "I am able to identify a small relevant Isolated System for some process and quantity."
- "I am able to read a motion diagram and describe the motion of a system."
- "I am able to use a crazy idea to predict the outcome of a testing experiment."
- "I am able to judge a crazy idea by comparing a prediction with the real outcome of a testing experiment."
- "I am able to translate force arrows into possible measurements and apply Newton's 2nd Law to determine the acceleration of a system."

### Writing Style for Skill Statements

- Always start with "I am able to..."
- Be specific about what the student does (read, draw, identify, analyze, predict, judge, translate, apply)
- Reference the specific physics representations or tools involved (bar charts, motion diagrams, force diagrams, isolated systems, etc.)
- Keep the statement to one sentence

## Level Descriptor Patterns

Each level follows a specific writing pattern using structured connectors: **-BUT-**, **-BECAUSE-**, **-AND-**, **-OR-**

These connectors are always written on their own line, in bold, surrounded by hyphens.

### Missing (0%)

Pattern: "There is no attempt to [skill action]."

Always a single short sentence. Examples:
- "There is no attempt to read the bar chart."
- "There is no attempt to identify an Isolated System."
- "There is no attempt to make a prediction."
- "There is no attempt to draw a bar chart."

### Emerging (55%)

Pattern: "There is [some/an] attempt to [skill action] **-BUT-** [critical failure] **-OR-** [alternative failure]"

The student tried, but the attempt has fundamental problems. Use -BUT- to introduce the critical gap, then -OR- to list alternative ways the attempt falls short.

Example:
```
There is an attempt to read the bar chart
-BUT-
The response is not relevant to the task.
-OR-
There is no explanation supporting the chosen system.
```

Example:
```
There is some attempt to draw a bar chart.
-BUT-
The attempt does not have the correct format for a bar chart.
-OR-
The bar chart is tracking the wrong quantity.
```

### Approaching (65%)

Pattern: "There is a decent attempt to [skill action] **-BECAUSE-** [what's correct] **-BUT-** [what's still wrong] **-OR-** [alternative remaining issue]"

The student shows some competence (-BECAUSE- explains what's right), but significant issues remain (-BUT- explains what's wrong).

Example:
```
There is a decent attempt to identify an Isolated System
-BECAUSE-
The system is relevant to the process and what is being measured.
-BUT-
The system is not an isolated system for what is being measured during the process.
-OR-
An isolated system is identified, but there is minimal explanation supporting the chosen system.
```

### Developing (85%)

Pattern: "There is a good attempt to [skill action] **-BECAUSE-** [what's correct] **-AND-** [additional correct element] **-BUT-** [minor issues remaining]"

Most criteria are met (-BECAUSE- and -AND- list what's correct), with only minor issues remaining (-BUT-).

Example:
```
There is a good attempt to identify an Isolated System
-BECAUSE-
The system is relevant to the process and what is being measured.
-AND-
The system is an isolated system for what is being measured during the process.
-BUT-
There is an incomplete explanation supporting the chosen system.
-OR-
The system is not the smallest isolated system.
```

### Refining (100%)

Pattern: "There is an excellent attempt to [skill action] **-BECAUSE-** [correct] **-AND-** [correct] **-AND-** [mastery indicator]"

All criteria are met. Only uses -BECAUSE- and -AND- (no -BUT-). The final -AND- describes the mastery indicator that distinguishes Refining from Developing.

Example:
```
There is an excellent attempt to identify an Isolated System
-BECAUSE-
The system is relevant to the process and what is being measured.
-AND-
The system is an isolated system for what is being measured during the process.
-AND-
There is an excellent explanation supporting the chosen system.
-AND-
The system is the smallest isolated system.
```

---

## Key Conventions

1. **Progressive building:** Each level builds on the previous one. Developing includes everything from Approaching plus more. Refining includes everything from Developing plus mastery.

2. **Connectors on their own line:** `-BUT-`, `-BECAUSE-`, `-AND-`, `-OR-` always appear on their own line, not inline within a sentence.

3. **Concrete examples:** When possible, include "For example" bullet lists showing specific ways a criterion might or might not be met (especially at the Approaching and Developing levels).

4. **Qualitative progression of attempt adjectives:**
   - Emerging: "some attempt" or "an attempt"
   - Approaching: "a decent attempt"
   - Developing: "a good attempt"
   - Refining: "an excellent attempt" or "[skill] is expertly [done]"

5. **Multiple skills per question:** A single assessment question typically evaluates 2-4 distinct skills. Each skill gets its own row in the rubric table.

6. **Skill specificity:** Rubric criteria must be specific to the actual physics content being assessed. Generic criteria like "shows understanding" are insufficient — specify what understanding looks like for this particular skill and concept.

## Example: Complete Rubric Table

**Assessment Question: Reading Motion and Force Diagrams**

| Skill | Missing 0% | Emerging 55% | Approaching 65% | Developing 85% | Refining 100% |
|-------|-----------|-------------|-----------------|----------------|---------------|
| I am able to read a motion diagram and describe the motion of a system. | There is no attempt to write a story trying to match the motion diagram. | A story is written describing a moving system whose motion tries to match the motion diagram -BUT- The story is difficult to understand. -OR- The motion of the system of interest is very different from the motion described by the motion diagram. | A story is written describing a moving system whose motion is beginning to match the motion diagram -BUT- The direction of the system of interest's motion does not match the direction of the velocity arrows in the motion diagram. -OR- The system of interest's motion does not match the length of the velocity arrows. | A story is written describing a moving system whose motion matches the motion diagram. -AND- The system of interest's motion matches the velocity arrows. -AND- The system of interest's motion matches the change in velocity arrows. | A story is written describing a moving system whose motion matches the motion diagram. -AND- The system of interest's motion matches the velocity arrows. -AND- The system of interest's motion matches the change in velocity arrows. -AND- The motion is described either like a scientist or a mathematician. |
